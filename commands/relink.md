# /slotmachine:relink — 링크 재판단

vault에 새 문서가 추가되거나 기존 문서가 변경되었을 때, 누락된 위키링크를 탐색하고 삽입한다.

## 동작 방식

- **Delta 모드 (기본)**: `links_evaluated_at` 기준으로 미평가 또는 변경된 문서를 피벗으로 선별해 탐색한다.
- **Full 모드 (`--all`)**: Archives를 제외한 전체 vault를 대상으로 탐색한다.
- Archives 카테고리는 피벗 및 후보에서 자동 제외된다.
- 임베딩 API 호출 없음 — GraphDB에 저장된 벡터를 재사용한다.

## 실행 순서

### 1단계 — 모드 확인

사용자 입력에서 모드를 결정한다.

- 인수 없음 → `mode: "delta"`
- `--all` 포함 → `mode: "all"`
- `--para=Projects,Resources` 형태 → `para_filter: ["Projects", "Resources"]`

Full 모드인 경우, 실행 전 안내 메시지를 출력한다:
```
⚠️ Full 모드: Archives 제외 전체 vault를 대상으로 링크를 재판단합니다.
문서 수에 따라 수 분이 소요될 수 있습니다. 계속할까요? [Y/N]
```
사용자가 Y를 입력한 경우에만 진행한다.

### 2단계 — relink 툴 호출

```json
{
  "mode": "delta",
  "para_filter": null
}
```

반환값:
- `pivots_found`: 피벗 문서 수
- `pivots_with_candidates`: 후보가 발견된 피벗 수
- `results`: 피벗별 후보 목록

### 3단계 — 결과 표시

후보가 있는 피벗 문서별로 테이블을 표시한다.

```
🔗 링크 재판단 결과 (delta 모드)
피벗 N개 중 M개에서 새 링크 후보 발견

──────────────────────────────────────────────────────────────
 #  피벗 문서                    후보 문서                카테고리  점수
──────────────────────────────────────────────────────────────
 1  새_프로젝트_기획.md          클린코드_요약             Resources 0.82
                                아키텍처_패턴_노트         Resources 0.76
 2  독서노트_원칙.md             2024_리팩토링_프로젝트    Projects  0.71
──────────────────────────────────────────────────────────────

승인하시겠습니까?
[전체 승인: Y / 개별 선택: E / 취소: N]
```

후보가 없으면 "재판단 대상이 없습니다. vault가 최신 상태입니다." 라고 알리고 중단한다.

### 4단계 — 링크 삽입

사용자 승인 후 피벗 문서별로 `apply_links`를 호출한다.

```json
{
  "path": "피벗 문서 경로",
  "link_titles": ["후보1", "후보2"]
}
```

`apply_links`가 성공하면 해당 문서의 `links_evaluated_at`이 자동으로 갱신된다.

### 5단계 — 완료 보고

```
✅ 링크 재판단 완료.
  - 새_프로젝트_기획.md: [[클린코드_요약]], [[아키텍처_패턴_노트]] 삽입
  - 독서노트_원칙.md: [[2024_리팩토링_프로젝트]] 삽입

다음 단계: /slotmachine:save 로 원격 저장소에 push하세요.
```

오류가 있으면 오류 내용을 함께 출력한다.
